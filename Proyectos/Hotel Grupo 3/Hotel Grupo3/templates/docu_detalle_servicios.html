Detalles Servicios

Métodos para registrar los detalles del servicio que adquirio el huesped en el hotel.

Obtener todos los detalles servicios

Este método te permite obtener todos los detalles de servicion registrados en el hotel.

HTTP Request (GET) https://grupo3.pythonanywhere.com/api_obtenerdetalleservicios
No es necesario preparar un cuerpo en la solicitud
RESPONSE
[
{
"descripcion_solicitud": "Oferta de helados",
"fecha_solicitud": "2023-05-30",
"hora_solicitud": "11:30:00",
"id": 2,
"monto_servicio": "10.00",
"servicio_id": "Vitrina",
"transaccion_id": 1
},
{
"descripcion_solicitud": "Oferta brownis",
"fecha_solicitud": "2023-06-10",
"hora_solicitud": "12:30:00",
"id": 5,
"monto_servicio": "10.00",
"servicio_id": "Vitrina",
"transaccion_id": 1
}
]		 El cuerpo de respuesta, manifiesta todos los elementos pedidos con siete campos, a través de un JSON array.
Donde
descripcion_solicitud: Es la breve explicación escrita de las características, beneficios y cualidades del servicio.
fecha_solicitud: Es la fecha en la que se solicitó el servicio.
hora_solicitud: Es la hora en la que se solicitó el servicio.
id: Es la identificación del detalle servicio.
monto_servicio: Es la suma final del servicio brindado.
servicio_id: Es el nombre del servicio que se va a detallar.
transaccion_id: Es la identificación de la transacción del huésped.

Guardar detalle de servicio

Este método te permite crear un detalle servicio nuevo.

HTTP Request (POST) https://grupo3.pythonanywhere.com/api_guardardetalleservicios
{ "fecha_solicitud" : "2023-06-10",
"hora_solicitud" : "12:30:00",
"descripcion_solicitud" : "Oferta brownis",
"monto_servicio" : "10",
"transaccion_id": 1,
"servicio_id":3
}		 El cuerpo de solicitud, manifiesta todos los elementos a pedir con seis campos, a través de un JSON array. Aqui no se coloca el campo id, el sistema se encargará de asignarle el id correspondiente.
Donde
fecha_solicitud: Es la fecha en la que se solicitó el servicio.
hora_solicitud: Es la hora en la que se solicitó el servicio.
descripcion_solicitud: Es la breve explicación escrita de las características, beneficios y cualidades del servicio.
monto_servicio: Es la suma final del servicio brindado.
servicio_id: Es la identificación del servicio que se va a detallar.
transaccion_id: Es la identificación de la transacción del huésped.
RESPONSE
{ "Mensaje": "Detalles de servicio registrado correctamente" } El cuerpo de respuesta, manifiesta que el detalle de servicio ha sido registrado correctamente. Donde Mensaje: Es la respuesta de la lógica de negocio de detalle servivio.
