Comprobante

Métodos para registrar los comprobantes que adquirio el huesped en el hotel.

Obtener todos los comprobantes

Este método te permite obtener todos los comprobantes registrados en el hotel.

HTTP Request (GET) https://grupo3.pythonanywhere.com/comprobante
No es necesario preparar un cuerpo en la solicitud
RESPONSE
[
{
"fecha_comprobante": "21-06-2023",
"id": 1,
"monto_total": "400.00",
"numero_comprobante": "001-001",
"persona_id": 4,
"tipo_comprobante": "B",
"transaccion_id": 1
},
{ "fecha_comprobante": "22-06-2023",
"id": 2,
"monto_total": "400.00",
"numero_comprobante": "001-002",
"persona_id": 8,
"tipo_comprobante": "F",
"transaccion_id": 2
}
]		 El cuerpo de respuesta, manifiesta todos los elementos pedidos con siete campos, a través de un JSON array.
Donde:
fecha_comprobante: Es una breve explicacion de la fecha del comprobante del dia que se genera o quieres reservar.
id: Viene hacer el id del comprobante.
monto_total: Es el monto que tu quieres o prefieres.
numero_comprobante: Es el codigo del comprobante.
persona_id: Es la foranea de persona.
tipo_comprobante: Es tipo de comprobante que elijes, B (Boleta),F (Factura), T (Ticket)
transaccion_id: Es la foranea de transaccion.

Guardar comprobante

Este método te permite crear un comprobante nuevo.

HTTP Request (POST) https://grupo3.pythonanywhere.com/api_guardarcomprobante
{ {
"fecha_comprobante": "25-06-2023",
"id": 3,
"monto_total": "100.00",
"numero_comprobante": "001-003",
"persona_id": 5,
"tipo_comprobante": "T",
"transaccion_id": 3
}
}		 El cuerpo de solicitud, manifiesta todos los elementos a pedir con seis campos, a través de un JSON array. Aqui no se coloca el campo id, el sistema se encargará de asignarle el id correspondiente.
Donde:
fecha_comprobante: Es una breve explicacion de la fecha del comprobante del dia que se genera o quieres reservar.
id: Viene hacer el id del comprobante.
monto_total: Es el monto que tu quieres o prefieres.
numero_comprobante: Es el codigo del comprobante.
persona_id: Es la foranea de persona.
tipo_comprobante: Es tipo de comprobante que elijes, B (Boleta),F (Factura), T (Ticket)
transaccion_id: Es la foranea de transaccion.
RESPONSE
{ "Mensaje": "Comprobante registrado correctamente" } El cuerpo de respuesta, manifiesta que el comprobante ha sido registrado correctamente. Donde Mensaje: Es la respuesta de la lógica de negocio de comprobante.
